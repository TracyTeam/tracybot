# Opencode Plugin API Findings

## Event Types Available

### Session Events
- `session.created` - New session starts
- `session.updated` - Session properties updated
- `session.deleted` - Session deleted
- `session.idle` - Session became idle
- `session.status` - Session status changed
- `session.compacted` - Session was compacted
- `session.error` - Session had an error
- `session.diff` - Session diff available

### Message Events
- `message.updated` - Message metadata updated
- `message.removed` - Message removed
- `message.part.updated` - Part content updated (used currently)
- `message.part.delta` - Part delta update (more granular)
- `message.part.removed` - Part removed

### Other Relevant Events
- `tool.execute.before` - Tool about to execute (used currently)
- `tool.execute.after` - Tool finished (used currently)
- `command.executed` - Shell command executed

## Key Finding: Message Completion Detection

### Option 1: Use `finish` field on AssistantMessage
From `types.gen.d.ts:178-211`:
```typescript
export type AssistantMessage = {
  // ...
  time: {
    created: number;
    completed?: number;  // timestamp when completed
  };
  finish?: string;  // "stop" | "length" | "content_filter" | etc.
  // ...
}
```
The `finish` field indicates why the response ended.

### Option 2: Use `experimental.text.complete` hook
From `index.d.ts:229-235`:
```typescript
"experimental.text.complete"?: (input: {
    sessionID: string;
    messageID: string;
    partID: string;
}, output: {
    text: string;
}) => Promise<void>;
```
This hook fires specifically when text generation completes!

## Current Tasklet Creation Issue

Current flow (index.ts:142-157):
1. `message.part.updated` fires on EVERY text chunk during streaming
2. Each chunk triggers `createTasklet()` 
3. Results in duplicate/incomplete tasklets

## Recommended Fix

Use `experimental.text.complete` hook instead of checking in `message.part.updated`:

```typescript
"experimental.text.complete": async (input, output) => {
    const { messageID } = input
    const msg = messages.get(messageID)
    if (msg && msg.role === "assistant" && msg.agent === "build") {
        // Create tasklet here - text is complete
        createTasklet(currentSessionId, {
            ...msg,
            text: output.text
        })
    }
}
```

## Alternative: Filter by `finish` field

In `message.updated` event, check if the message has `finish` set:
```typescript
if (event.type === "message.updated" && event.properties?.info) {
    const info = event.properties.info
    if (info.role === "assistant" && info.finish) {
        // Message is complete
    }
}
```
